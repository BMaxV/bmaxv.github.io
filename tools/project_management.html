Project Management

These tools are motivated by me encountering problems and not being able to deal with them effectively. Be it because the necessary tool didn't exist or because I didn't have access to it.

I will probably release them as open source eventually, because I really just want everyone to not have these problems anymore. Sooner or later they're bound to be my problem as well. Be it by others spending "opportunity cost" time on it, or because not using them, or tools like them, actively creates problems.

dependency management

Suppose there is a chain of people working on something and they had over stuff.

And supposed A notices he made a mistake, but it's already in C's hands. Multiple things should be possible:

  • informing stakeholders "upstream" of additional work to be done (red)
  • informing stakeholders "downstream" of potential delays and issues (blue)

This becomes more relevant the bigger the dependency tree.

That should be a very very simple interaction. Of course that depends there being an organization chart for the project and stakeholders. What I have handles the network aspect, but not the actual message sending, that's up to the user of the library.

I have not seen a tool to do this effectively out in the wild, yet.
