technical drawings

I think the format to do technical drawings in should be svg/html for these reasons:

  • it's text based
  • it's machine readable
  • it's easy to edit manually
  • it supports links and comments
  • it's printable
  • it's supported by virtually every device
  • it has a very small memory footprint
  • it's supported by lots and lots of software, for editing, viewing and analysis
  • it supports "real" measurements such cm, mm, directly
  • it supports every type of line or filling styles you could want
  • it supports grouping and inheritance
  • you can extend it in any way you want

As long as you want something to be 2d, that should tick all boxes.

That anyone uses other formats, or even pays for it, is so weird!
