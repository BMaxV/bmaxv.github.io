year of the linux desktop

There is a meme about the year of the linux desktop, and everyone always says, "this is the year" and it's bullshit every time. Here are some reasons from a user perspective of why that won't happen with the current attitude. In no particular order, but maybe I can put the important ones at the top.

I'm not accusing anyone. I'm not demanding that this be fixed. I am merely stating that these things are the difference, between me, a somewhat technically minded person being able to plopp a linux distro on my grandpa's PC and tell him "just look for the help button and go from there". ...and not being able to do that.

The difference can be a simple "let me google that for you" button. It can be a manual, but then the manual should be complete and translated, which is always tough. It can also be a disclaimer that says "look, this is experimental, thanks for using it, if you have problems write me an email at ... or leave message on github/mastodon/facebook/linkedin".

The things it should not be: obscure or require meta knowledge to even find. This applies to console commands as well. "ls" and "cd" work fine, but if the target person hasn't receieved some kind of hand holdy experience as an introduction to linux or the command line, they will be helpless here is as well.

What is making me slightly angry is the sheer arrogance:

"my software is so perfect, so obvious, the user will never need help, there will never be a bug, so there doesn't need to be a bug report button. It is so obviously the most bestest software, that the user doesn't need to know who made it, how to turn it off, and they most certainly will never need to know how it works. As long as this attitude persists, I will recommend linux begrudgingly, as a lesser evil. If someone complains that linux is not for them, I won't correct them. Because this stuff sucks, it's a problem and they may be completely right.

The help/bug report/time to fix is completely unpredictable

let's take this as a source for some distros https://distrowatch.com/

Let's say you do find a bug. Let's say you even find the organisation who made the software, the correct forum or bug tracker or whatever. None of the major bug tracking system have any kind of metric to judge how fast you can expect help.

  • launchpad tells you the latest questions like it's facebook. "Look what xyz is up to". Search is package name based, so GOOD LUCK.
  • the kde bug tracker is better, but not ideal. At first sight, it's simple at least.
  • open suse tells non technical users to try the forum first.
    which is sort of fine I suppose, except I don't think open suse will have subforums and topics for every package. It would be unreasonable to expect and frankly, this already incentivizing a bad data flow:
    • non technical users phrase their question as text
    • experienced users have to parse the text and find the appropriate data link / package / etc.
    • free text has a good chance that it doesn't contain the relevant info like logs, system setup info. They have probably have to be asked for everytime.
    In short, it would be much, much simpler if the operating system had a general "help" button that would already collect this info. But alas.
  • fedora has general Q\&A forum, and a live chat that you have to sign up for, which I think is relatively fair.
    their "get fedora" page has no link to filing bugs. The docs have a friendly little box saying:
    Anyone can file bugs: All users are encouraged to file any bugs they run into. Bug filing is not limited to only software developers.
    Which is an arrogant, bold faced lie. You dont' want that. Regular users don't want that either.

    The irony is alive and well of course, because the rest of this docs page is pretty good. In fact, it's good enough to be a general guide to reporting bugs anywhere, excluding some platform specific individual advice and how to's about using e.g. bugzilla.

  • linux mint does not have a "help" button. If you look for an issue tracker, you will probably find this page which will basically redirect to github. Which is sort of fine?

Basically, this is developer land.

Which is fine, of course.

But you really shouldn't be surprised that a normal user (read hank hill), being foreign to it, will feel excluded and not wanting to parcitipate in it.
